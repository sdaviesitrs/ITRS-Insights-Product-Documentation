Aggregation

An aggregation is a Select clause where all fields are either values in a group or values produced by an aggregator function.

Form

FROM HISTORICAL <stream_name>
[,<query>]?
SELECT <, group field | aggregator function>

For example:

FROM HISTORICAL {{book.cpu}}
SELECT avg(user) AS user, avg(kernel) AS kernel

Rules:

  • An aggregated select must contain aggregator functions for all selected fields (that aren’t group fields in a Group By) - i.e. a select clause cannot mix aggregated and non-aggregated results.

The following is invalid as user AS is not an aggregation. Nor is user + kernel AS sum an aggregation.

FROM {{book.cpu}} SELECT user AS u, avg(user) AS user, user + kernel AS sum

The behaviour of an aggregation depends on the time context the query will be

  • A purely historical <insights-syntax-aggregation-HistAggregation> aggregation generates a single output.
  • A purely realtime <insights-syntax-aggregation-RealTimeAggregation> aggregation will emit an output for each input.
  • A bootstrapped <insights-syntax-aggregation-BootAggregation> aggregation will emit a single output for the historical results and then an output for each realtime input received subsequently.

Historical Aggregation

A historical aggregation is an aggregation on purely historical data. A single output will be generated from the query.

For example:

FROM HISTORICAL {{book.cpu}}
SELECT avg(user) AS user, avg(kernel) AS kernel

If the input was:

{ "user": 10, "kernel": 5 }
{ "user": 20, "kernel": 8 }
{ "user": 9, "kernel": 11 }
The result would be:
{ "user": 13, "kernel": 8 }

Realtime Aggregation

A realtime aggregation is an aggregation on purely realtime data. Output will be generated for each input payload. For example:

FROM {{book.cpu}}
SELECT avg(user) AS user, avg(kernel) AS kernel

If the input was:

{ "user": 10, "kernel": 5 }
{ "user": 20, "kernel": 8 }
{ "user": 9, "kernel": 11 }

then the result would be :

{ "user": 10, "kernel" 5 }
{ "user": 15, "kernel": 6.5 }
{ "user": 13, "kernel": 8 }

Bootstrapped Aggregation

A bootstrapped aggregation is an aggregation that starts off processing historical data and then switches across to realtime input. For example

FROM {{book.cpu}}
SELECT avg(user) AS user, avg(kernel) AS kernel
If what is already stored is:
{ "user": 10, "kernel": 5 } { "user": 20, "kernel": 8 }
Then the result will be:
{ "user": 10, "kernel" 5 }
If you add the new input:
{ "user": 9, "kernel": 11 }
then the new result would be:
{ "user": 13, "kernel": 8 }

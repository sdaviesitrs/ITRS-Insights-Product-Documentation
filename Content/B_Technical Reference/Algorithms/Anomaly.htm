Anomaly

Identification of events which do not conform to the pattern set by other events in a dataset.

This is done using the SAX (Symbolic Aggregate ApproXimation) method described in greater detail in the paper "Assumption-Free Anomaly Detection in Time Series". Using this method the time series is converted into a sequence of symbols or 'word', depending in which zone that section of the series lands. This representation of the series as a string can then be used for anomaly detection with high precision over a wide variety of datasets.

Format

anomaly(lagWindow, leadWindow, featureSize)(value:double)

Parameters

Name Type Meaning
lagWindow Int How much memory of the past you use to judge the future. Lag window size is how much memory the algorithm should keep to compare to the data in the lead window
leadWindow Int Lead window size is how far to look ahead. A reasonable value would be 2 or 3 times the length of feature window size
featureSize (default 5) Int Feature window size should be about the size at which events happen

Rough guidance is:

  • featureSize * 3 = leadWindow
  • leadWindow/2 = lagWindow

featureSize:

Experience has shown that processing a long time series by creating one single SAX word for representation does not produce such good results and so we create a shorter window over the series (called a feature window) which then slides accross the time series, we can then create a set of shorter SAX words to process. The featureSize parameter is the length of this feature window. The feature window size should be about the size at which events happen, so for instance if analysing a heart rate this should be the length of a heart beat. If unspecified this defaults to 5.

Examples:

from /streams/demo/infrastructure/ecg
group by contributor.building  as b
select b, anomaly(lagWindow = 200, leadWindow = 40, featureSize=5)(value) as a
emit every value

The wordLength relates the granularity of the detail captured, and defines how many characters the SAX word should have, i.e. how many chunks should the series be broken in to. In the example diagram below the wordLength is 8 for clarity, however 3 is considered a reasonable number for many applications, thus it is our default.

Algorithms based on the paper:

"Assumption-Free Anomaly Detection in Time Series"
By: Li Wei, Nitin Kumar, Venkata Lolla, Eamonn Keogh, Stefano Lonardi & Chotirat Ann Ratanamahatana, Department of Computer Science & Engineering, University of California - Riverside

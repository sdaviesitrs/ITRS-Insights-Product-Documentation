Gateways

Valo can connect to one or more publishing Gateways.

In application.conf, valo.transport.geneos.gateways is an array of Gateway nanomsg connections the node should connect to. Each gateway has the following settings:

Setting Description
client The client to which gateway publishes the data. Either nanomsg or kakfa
host The host/ip of the gateway
polling-interval

The rate at which the gateway will polled for changes. If omitted the default-gateway-polling-interval will be used (See Configuration)
sub-port The subscription port. Defaults to 7070
req-port The request port. Defaults to 7071

To connect to a single gateway publishing to nanomsg using the default ports:

    {
        "valo.transport.geneos.gateways": [
                    {  "host" : "myhost" }
        ]
    }

Or multiple gateways:

    {
        "valo.transport.geneos.gateways": [
            {  "host" : "myhost1" },
            {  "host" : "myhost2" },
            {  "host" : "myhost3" }
        ]
    }

Ports and Polling

Each nanomsg publishing Gateway has 2 ports for publishing data to Valo, the request port and the subscribe port.

The request port (default 7071) is used by the node to request meta-data from the Gateway, such as the version, uptime, lists of probes, dataviews etc. The node will poll the Gateway at the polling interval, checking for changes in the Gateway state.

The subscribe port (default 7070) is used to subscribe to data in the Gateways using a Publish/Subscribe pattern.

The ports and the polling interval can be set per gateway:

    {
        "valo.transport.geneos.gateways": [
            {
                "host" : "myhost",
                "sub-port" : 50200,
                "req-port" : 50201,
                "polling-interval" : "1.minute"
            }
        ]
    }

Hot Standby

Valo Geneos integration also supports hot standby Gateway pairs. In the event that the primary Gateway fails, the secondary will take over and start publishing the data to Valo

    {
        "valo.transport.geneos.gateways": [
            {
                "host" : "primaryhost",
                "secondary" : {
                    "host" : "secondaryhost"
                }
            }
        ]
    }

The secondary can have all the same settings as the primary:

    {
        "valo.transport.geneos.gateways": [
            {
                "host" : "primaryhost",
                "secondary" : {
                    "host" : "secondaryhost"
                    "sub-port" : 50200,
                    "req-port" : 50201,
                    "polling-interval" : "1.minute"
                }
            }
        ]
    }

Introduction

 

Notebooks

So in our smart street example if I want to analyze the energy usage in house number 4 then I open a notebook and select House04, I am then able to perform searches and queries against that domain, from a simplistic search that requests how many times ‘kettle’ appears in that stream (that may represent every time a kettle is turned on and off) over a given time frame to complex queries like what is the combined electricity usage of all the smart appliances in the house from 6pm until midnight, displaying the results in a choice of graphical or tabular ways in the notebook.

Because ITRS Insights also has a time series repository, which means it understands to some extend the concepts of time, and also comes with some complex algorithms built into the query language then it is also possible to run a single query that is sophisticated enough to spot anomalies in the data.

So for instance if aging Mrs Brown at No 4 always watches her favorite soap at 7pm every evening and straight afterward gets up and makes a cup of tea then the notebook can help you spot that this evening she didn’t switch the kettle on at 8pm, which may be cause for concern. Because it is real-time then the queries can immediately identify anomalies, however you can also run the same thing against historic data to find previous patterns and areas for further investigation.

As with a paper notebook, these investigations and discoveries can be saved and shared with other people and added to; and unlike paper they can also be rerun with just the touch of a button.

Dashboard

Dashboards provide an aggregated view of your visualizations to enable you to see the results of key in real-time. Once you've created one or more queries or free text searches in a notebook, important results and visualizations can be sent to a dashboard where changes can be seen in real-time.

For more about visualizing your results, see [%=General.Product Name Insights (Full)%] Visualizing Results User Guide.

You can drag and drop the visualizations in a Dashboard, allowing you to order the space as you wish. In addition, you can resize each of the visualizations to increase or decrease their prominence in the view - when you do so, other visualization panels in the Dashboard will resize accordingly.

Your panel selections, positions and sizes are saved automatically.

Working With Dashboards

To create a dashboard, select the dashboard icon from the menu on the left hand side of the home page (or in fact any page).

 

The Notebooks menu will pop out, to create a new one then hit the + button.
This creates a new dashboard for you with a default name. If you want to change the name of the dashboard them select the edit icon edit the dashboard details as you choose

and then save the changes using the button at the bottom left of the screen

Your dashboard is now ready to send query results to.

Populating A Dashboard

Once you have a dashboard then you need to select which queries from your :doc:`notebook <working_with_notebooks>` you wish to publish there.

When you have run a query in a notebook and have either a table or selected a visualisation you can send that visualisation to a dashboard by selecting the Send To Dashboard icon src="../Resources/Images/ from the top right of the query panel. This passes you into the Send to dashboard wizard where you can specify which of your already created dashboards you wish to send the query results to

then configure the visualisation by giving it a name

and previewing what will appear on the dashboard before select to send it. You will receive a brief message on the screen to confirm it has been sent and to where.

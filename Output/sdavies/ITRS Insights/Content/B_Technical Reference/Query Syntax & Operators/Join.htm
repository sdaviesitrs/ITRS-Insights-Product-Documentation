Join

Joins two streams.

The stream to join to can be specified inline or another query in the same session can be referenced by name.

Form:

<query> [INTO <name>]?
INNER JOIN <name|query> ON <predicate>

*Note:*The use of what may appear at the moment to be a superfluous INNER operator is deliberate because in the future it is expected others options will be available.

Examples:

Join two streams directly

from historical /streams/demo/infrastructure/cpu
    inner join from /streams/demo/infrastructure/mem
        on /streams/demo/infrastructure/cpu.host == /streams/demo/infrastructure/mem.host
select /streams/demo/infrastructure/cpu.host, /streams/demo/infrastructure/cpu.user, /streams/demo/infrastructure/mem.kernel

In practise aliasing the stream names makes this much easier to read, so for example

from historical /streams/demo/infrastructure/cpu into left
    inner join from /streams/demo/infrastructure/mem into right
    on left.host == right.host
select left.host, left.user, right.kernel

Joins can appear anywhere in the query.In this case, a predicate filters the data flowing into the left side of the join

from historical /streams/demo/infrastructure/cpu
where user > 50
into left
inner join from historical /streams/demo/infrastructure/mem
    into right
    on left.host == right.host
select left.host, left.user, right.kernel

The right hand side of the join can contain further query expressions, in this case a simple WHERE predicate

from historical cpu
inner join from historical mem
    where kernel > 40.0
on cpu.host == mem.host
select left.host, left.user, right.kernel

Joins can be combined with time windows, calculating the join once the window closes and allowing operation across unbounded streams. In the example below, the window is 10 seconds on both sides of the join and we use the same field in both streams.

from cpu
    inner join from mem
    on cpu.host == mem.host
    group by sampleTime window of 10 seconds
select cpu.host, cpu.user, mem.kernel

As above, but using different fields for creating the windows on either side

from cpu
    inner join from mem
        on cpu.host == mem.host
        group by sampleTime, timestamp window of 10 seconds
select cpu.host, cpu.user, mem.kernel

The size of the window can be configured independently on each side of the join

from cpu
 inner join from mem
     on cpu.host == mem.host
    group by sampleTime window of 10 seconds, timestamp window of 60 seconds
select cpu.host, cpu.user, mem.kernel
